Predicted values are generated on the test set, and have (expX – 1) applied to invert the log(1+x) on the dependent variables.

1.       OLS Linear Regression

a.       LinearRegression fits a linear model with coefficients for each feature to minimize the residual sum of squares between observed targets and predicted targets

b.       No regularization

2.       Linear regression with restricted features

a.       I use Recursive Feature Elimination to reduce the number of variables to a limited set that are most relevant in predicting the target variable

b.       hyperparameter tuning on n_features_to_select through GridSearchCV

3.       LASSO

a.       Model penalized for sum of absolute values of weights, so many features will be weighted to 0.

d.       will have to do some hyperparameter tuning on alpha through GridSearchCV

4.       Exposition, problem description, and management recommendations

The research question of interest was to find good prediction of house sale price when provided with its features. The models used in this week’s assignment was able to get a decent score but it seems like it will be difficult to get below 0.10. Stacking multiple regression models via a meta-regressor will probably be necessary.

Comparing my models show me that most of the features influence the target Sale Price in some way, but many of them likely have very small weights.

For management recommendations,  obviously a bigger data set will result in better models. When split by dwelling class or neighborhood, there becomes very few samples in each class.

 

 

Appendix 2: Data preprocessing

b.       'Utilities' - Only 1 unit does not have all utilities.

a.       'MSSubClass' - int64 but housing type is a categorical variable

b.       'YrSold' - int64 but year is categorical

c.       Replacing features with few missing values with the most frequent value: 'MSZoning', 'Electrical', 'KitchenQual', 'Exterior1st', 'Exterior2nd', 'SaleType', 'Functional'

d.       Replace ‘LotFrontage’ NaN values with the median of other ‘LotFrontage’ values in its neighborhood

4.       Outlier removal

a.       Using a quantile method removing only rows with extreme values vs SalePrice

b.       Will revise with a manual method based on feature vs target plots

5.       Removing multicollinear features

a.       Features best correlated with ‘SalePrice’ are ‘OverallQual’, ‘GrLivArea’, ‘GarageCars’, ‘GarageArea’, ‘TotalBsmtSF’, ‘1stFlrSF’, ‘FullBath’, ‘TotRmsAbvGrd’, and ‘YearBuilt’

b.       Features that exhibit multicollinearity are ‘TotalBsmtSF’, ‘GrLivArea’, ‘LowQualFinSF’, ‘2ndFlrSF’, ‘1stFlrSF’, ‘BsmtFinSF1’, ‘BsmtFinSF2’,  and ‘BsmtUnfSF’.
